# 🧭 Crypto Network Guard

Crypto Network Guard is a prototype designed to help users avoid mistakes when sending cryptocurrency across blockchain networks.

Many cryptocurrencies share similar address formats. For example, multiple networks use the same **0x-style EVM address**, which can lead users to select the wrong network when sending assets. Sending funds through the wrong network may result in lost or inaccessible funds.

This prototype combines **address detection, risk analysis, AI explanations, and exchange market data** to help users make safer decisions.

---

# 🚀 Features

## Address Format Detection

The system detects the **format of the destination address**, not the token itself.

Supported formats:

- Bitcoin — addresses starting with `1`, `3`, or `bc1`
- TRON — addresses starting with `T`
- EVM networks — addresses starting with `0x`
- Solana-like — Base58 addresses (32–44 characters)
- BEP-2 — addresses starting with `bnb`

The application proposes compatible networks based on the detected format.

---

## Network Risk Assessment

After selecting a sending network, the system evaluates the **risk level**.

Low risk  
- Address format matches the selected network.

Medium risk  
- Address format is ambiguous (for example EVM addresses shared across networks).

High risk  
- Address format does not match the selected network.

The system also explains **why the risk level was assigned** and suggests a safer action.

---

## 🤖 AI Explanation Layer (LLM)

The prototype includes an **AI assistant powered by an LLM (Cohere API)**.

The LLM receives structured context generated by the application, including:

- detected address format  
- selected network  
- recipient type  
- risk level  
- risk explanation  
- recommended action  
- transfer goal  

Using this information, the model generates a **structured response** containing:

- headline  
- explanation  
- warning  
- suggested action  

This allows the prototype to translate system logic into a clear explanation for the user.

---

## 💰 Supported Recommendation Scenarios

To avoid misleading recommendations, the prototype only supports token-level interpretation for two controlled cases.

### Bitcoin
- interpreted as an investment-oriented asset  
- largest market capitalization  
- often drives market trends  

### USDT
- interpreted as a utility asset  
- mainly used for transfers and value storage  

For other ecosystems (for example Solana), the prototype performs risk checks but **does not generate token recommendations**.

---

## 🔗 BestChange API Integration

The application integrates the **BestChange API v2** to retrieve real exchange offers.

When supported scenarios are detected, users can load exchange offers directly from the BestChange monitor.

Offer sorting logic depends on the transfer goal:

Transfer  
- offers sorted by best rate  

Savings  
- offers sorted by lowest price  

This connects the prototype to **real market data** rather than theoretical recommendations.

---

# ⚙️ System Workflow

The application follows a sequential flow:

1. User pastes a crypto address  
2. The system detects the address format  
3. User selects the sending network  
4. The application evaluates the risk level  
5. User selects the transfer goal  
6. The AI assistant generates an explanation  
7. The system optionally loads BestChange offers  

The prototype therefore combines:

- rule-based validation  
- AI explanation  
- external market data
